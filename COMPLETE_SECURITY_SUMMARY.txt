================================================================================
                         SECURITY TOOLS INTEGRATED
================================================================================

TOOL 1: GITLEAKS (Secrets Scanning)
====================================
Purpose: Detect hardcoded secrets in git history
Detects:
  • API keys and tokens
  • AWS credentials
  • Database passwords
  • Private keys
  • OAuth tokens

Workflow: complete-security-scan.yml
Triggers: push, pull_request, daily schedule
Fails on: Any secrets found
Reports: SARIF format, PR comments

TOOL 2: OWASP DEPENDENCY CHECK (SCA)
====================================
Purpose: Scan dependencies for known vulnerabilities
Detects:
  • CVEs in packages
  • Outdated libraries
  • License issues
  • Security advisories

Workflow: security-scan.yml, full-security-pipeline.yml
Triggers: push, pull_request, daily schedule
Fails on: Critical vulnerabilities
Reports: JSON, HTML, SARIF formats

TOOL 3: SONARQUBE (SAST)
=======================
Purpose: Analyze code for vulnerabilities and quality issues
Detects:
  • Code smells
  • Vulnerabilities
  • Bugs
  • Security hotspots
  • Code coverage gaps

Workflow: sonarqube-scan.yml, security-scan.yml
Triggers: push, pull_request
Fails on: Quality gate violations
Reports: JSON, HTML, SARIF formats
Local: Docker Compose setup included

TOOL 4: TRIVY (Container Security)
==================================
Purpose: Scan Docker images and filesystems
Detects:
  • OS vulnerabilities
  • Application dependencies
  • Misconfigurations
  • Secrets in images

Workflow: full-security-pipeline.yml
Triggers: push, pull_request, daily schedule
Fails on: Critical vulnerabilities
Reports: SARIF, JSON formats
Integration: GitHub Security tab

TOOL 5: IAM HARDENING (AWS Security)
===================================
Purpose: Secure AWS deployments
Implements:
  • IAM roles instead of access keys
  • Least privilege policies
  • No hardcoded credentials
  • Environment variables
  • Temporary credentials (STS)

Workflow: deploy-aws.yml
Features:
  • OIDC federation with GitHub
  • No long-term credentials
  • Automatic credential rotation
  • Audit trail with CloudTrail
